自我介绍

对企业来说，我认为考察一个人应该是从两方面考察：第一方面，是这个人能做什么；第二方面，是这个人的成长空间，也就是软素质。所以，我分别从这两方面介绍自己：

现在能做什么

  • 优秀的前端

    • 熟练掌握 reactjs
      • 研二在北森云计算实习了7个月，熟练使用reactjs / immutablejs / redux / webpack
      • 负责了项目的部分重构。曾在北森述职答辩中获得了第一名
      • 了解 reactjs 的深层原理，曾自己实现了简易版的 react: my-reactjs
    • 熟练掌握 vuejs
      • 在网易使用 vuejs 一年时间，参与开发了一套 vue 组件库和对应的命令行工具： vusion
      • 使用 vuejs 完成过三个项目，积累了丰富的经验
    • 熟练掌握各种调试技巧。定位问题快速。

  • 优秀的 nodejs 工程师

    • 使用 nodejs 作为服务端语言开发过两个全栈项目，其中一个是 网易云域名系统
    • 在项目的过程中，对项目本身进行完善。解决了 配置管理 / 日志和错误处理 / 数据库版本控制 等痛点，并制作了一个模板初始化工具 uron-cli ，并从实践中抽离出两套模板 fullstack template / person template。
    • 自学并在 nodejs 端使用 graphql 作为接口规范，其中个人项目 react-native 应用 水果商城 的后端接口就是用 graphql 规范。
  • 掌握 react-native 应用
    • 有丰富的 reactjs 基础
    • 完成过一个 RN 应用 薛大人的水果铺。
  • 掌握 docker 并在实际项目中应用, 写过一篇 docker镜像制作及应用，是内部的一次分享的讲稿。
  • 有一定项目管理的经验：在网易现在负责两个项目，一个域名，一个内部运维系统。

软素质

  • 较强的自学能力
    • 从接触前端以来大概两年半，基本是遇到什么学什么，在保持精通 JavaScript 的基础上，学习过 gulp、webpack、php、nodejs、koa、mysql、mongoose、react、redux、immutable、vue、vuex、http 协议、restful api、graphql、fiddler、charles、linux、hexo 等等，也自己制作过扩展、制作过工具、搭过服务器、设计过数据库等，也深知贪多嚼不烂的道理，重要的会去了解原理，还有一些只是保证能完成工作即可。锻炼了一定的学习能力。
  • 一直表现优秀
    • 硕士表现优秀，我们学校2017届硕士毕业生代表。
    • 有学术研究能力，曾在《密码学报》上发表论文一篇《一种基于类名的网页大容量信息隐藏算法》。ps:《密码学报》是密码学方向国内最专业的期刊。所以，也有对技术的钻研精神。
    • 实习时用了不到三个月就熟练掌握了项目代码，负责项目的重构（六个实习生中唯一一个完全掌握的）。
    • 校招时才自学前端不到一年，拿到了网易的 special offer。
  • 不流于表面，对原理有探究精神
    • 为了解 react 和 regular，仿照正版写过 mini 版的 react 和 regular.
    • 深入阅读过《HTTP 权威指南》《你不知道的 JavaScript 》《ECMAScript 6入门》
    • 阅读过koa的源码
    • 阅读过nodejs中的源码，主要是http模块的实现，并自学过一些c++的知识。
  • 善于用程序解决问题
    • 为学习网络协议和代理的知识，做过一个冲顶大会助手 ，能拦截所有网络数据并抽取出 websocket 中问题，并自动打开浏览器进行搜索。
    • 做过识别验证码自动登录的 chrome 扩展
  • 总会想办法提高效率

项目经验

  • 域名系统 和 内部运维系统
    两个都是全栈项目，前端包括：vuejs、vuex、webpack，后端使用 koa / koa-router 等。从这两个项目里抽象出了全栈模板 fullstack

  • 后台管理系统
    纯前端项目，技术包括：vue-cli / vuejs / 高德地图 / vue-gaode。

  • 北森 cloud
    纯前端项目，技术包括： react / redux / immutablejs / webpack

  • 薛大人的水果铺
    全栈项目，前端使用 React-Native / mbox ，后端使用 koajs，前后端用 graphql 交互
