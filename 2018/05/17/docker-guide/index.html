制作的过程及使用方法

  • 用commit的方法

    用这种命令制作的镜像，在各个环境表现都是一样的，不会有环境的差异。

  • 用Dockerfile制作镜像

    一般用这种方式构建，原因如下：

    1，第一种构建的镜像不能重用

    2，第一种方式构建的镜像过程不透明，只能一层一层叠加，到最后可能使用者都忘记了这个镜像里面都有什么软件、什么环境，没法重新布置一套新的

    查看镜像历史 docker history t2/commit
